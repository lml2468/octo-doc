What it is

octo-doc is a self-hosted, Cloudflare-free way to publish prompt-native interactive HTML documents and collaborate on them. You publish a plain HTML file; octo-doc renders it, injects a thin review overlay, and gives every version a permanent URL.

It is one Go binary backed by PostgreSQL and any S3-compatible store — no vendor lock-in, no build step for your documents, and no JavaScript framework to learn. If it renders in a browser, it publishes.

How anchoring works

Comments in octo-doc are anchored to the text or artifact they refer to, not dumped in a flat thread at the bottom. Highlight a phrase and the comment sticks to those exact words. When you publish a new version, octo-doc re-anchors each comment to the same content — and clearly flags any comment whose text has changed so nothing silently drifts.

Publish in one call

Any HTML file becomes a living document with a single request:

curl -X POST "$OCTO/v1/docs" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"slug":"my-doc","title":"My Doc","html":"<h1>Hi</h1>"}'
# → { "data": { "version": 1, "url": ".../d/my-doc/v/1" } }

What you get out of the box

  • A permanent, shareable URL for every version
  • Anchored comments that survive re-publishing
  • Agent replies that mark each comment applied, partial, or a question
  • A distraction-free reading view with a review overlay on top

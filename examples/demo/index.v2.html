What it is

octo-doc is a self-hosted, Cloudflare-free way to publish prompt-native interactive HTML documents and collaborate on them. You publish a plain HTML file; octo-doc renders it, injects a thin review overlay, and gives every version a permanent, immutable URL that never changes underneath your readers.

It ships as a single Go binary backed by PostgreSQL and any S3-compatible object store — no vendor lock-in, no build step for your documents, and no front-end framework to learn. If it renders in a browser, it publishes.

How anchoring works

Comments in octo-doc are anchored to the text or artifact they refer to, not dumped in a flat thread at the bottom. Highlight a phrase and the comment sticks to those exact words. When you publish a new version, octo-doc re-anchors each comment to the same content — and clearly flags any comment whose text has changed so nothing silently drifts.

🔗 You're reading version 2. Comments left on version 1 have followed their anchors here. Where the underlying text changed, octo-doc marks the comment so a reviewer can re-anchor it deliberately.

Interactive artifacts

Documents aren't just prose. This adoption chart is a live, self-contained SVG artifact — the toggle redraws it with no page reload, and the whole figure is a single unit you can comment on. Version 2 adds a projected series alongside the actuals.

Versioning new in v2

Every publish creates a new immutable version. Old versions never change and keep their own URLs, so a link you shared last month still shows exactly what it showed then. The toolbar's version picker jumps between them, and a subtle strip appears when you're viewing anything other than the latest.

Immutability is a feature: reviewers argue about a specific version, not a moving target.

Publish in one call

Any HTML file becomes a living document with a single request:

curl -X POST "$OCTO/v1/docs" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"slug":"my-doc","title":"My Doc","html":"<h1>Hi</h1>"}'
# → { "data": { "version": 2, "url": ".../d/my-doc/v/2" } }

What you get out of the box

  • A permanent, shareable URL for every version
  • Anchored comments that survive re-publishing
  • Agent replies that mark each comment applied, partial, or a question
  • A distraction-free reading view with a review overlay on top
